# 架构详解

## 分层架构

```
┌────────────────────────────────────────────────┐
│                  AutoTest.Webapi                 │
│          Controller / SignalR Hub / 中间件        │
├────────────────────────────────────────────────┤
│             AutoTest.Infrastructure              │
│    仓储(Dapper) / HostedService / Outbox / AI    │
├────────────────────────────────────────────────┤
│              AutoTest.Application                │
│    Orchestrator / MonitorService / Pipeline     │
├────────────────────────────────────────────────┤
│  AutoTest.   AutoTest.   AutoTest.   AutoTest.  │
│  Execution   Assertions    Dsl     Orchestration│
├────────────────────────────────────────────────┤
│                  AutoTest.Core                   │
│       实体 / 接口 / 枚举 / DTO / 领域事件         │
└────────────────────────────────────────────────┘
```

## 各层职责

### Core — 领域层

纯 C# 类型，零外部依赖。定义：

| 类别 | 文件 | 说明 |
|------|------|------|
| 聚合根 | `Monitor/MonitorEntity.cs` | 监控任务实体，含状态机 |
| 聚合根 | `Execution/ExecutionRecord.cs` | 执行记录，含幂等键和心跳 |
| 目标 | `Target/Http/HttpTarget.cs` | HTTP 目标配置 |
| 目标 | `Target/Tcp/TcpTarget.cs` | TCP 目标配置 |
| 目标 | `Target/Db/DbTarget.cs` | DB 目标配置（SqlServer/MySQL/PG） |
| 目标 | `Target/Python/PythonTarget.cs` | Python 脚本目标 |
| 目标 | `Target/Template/TemplateTarget.cs` | DSL 模板目标（Type="TEMPLATE", DslJson） |
| 结果 | `Execution/ExecutionResult.cs` | 执行结果基类（IsExecutionSuccess, ErrorMessage, Assertions） |
| 断言 | `Assertion/AssertionResult.cs` | 断言结果 record（AssertionId, Target, IsSuccess, Actual, Expected, Message） |
| 断言 | `Assertion/AssertionRule.cs` | 断言规则（Id, Type, ConfigJson） |
| DSL | `Dsl/DagDefinition.cs` | DAG 定义：StepSequence → Items(StepDefinition|ParallelGroup) |
| DSL | `Dsl/DslPipelineContext.cs` | 运行时上下文：DslRuntimeContext, StepExecutionRecord, StepAssertionResult |
| 事件 | `Outbox/MonitorExecutionFailedPayload.cs` | 失败事件负载 |
| 仓储接口 | `Repositories/I*.cs` | IMonitorRepository, IExecutionRecordRepository, IOutboxRepository, IAnalysisRepository |
| 管线 | `ExecutionPipeline/IPipelineStep.cs` | 管线步骤接口 |
| 管线 | `ExecutionPipeline/PipelineContext.cs` | 管线上下文（Monitor, Result, Exception, Items） |
| 执行 | `Execution/IExecutionEngine.cs` | 执行引擎接口：CanExecute(MonitorTarget), ExecuteAsync |

### Application — 应用层

| 类 | 职责 |
|----|------|
| `Orchestrator` | 执行主流程：管线 → 断言 → 落库 → Outbox |
| `MonitorService` | 监控 CRUD、执行调度、缓存管理 |
| `Pipeline` | 管线实现，按反向顺序执行 IPipelineStep |
| `ExecutionStep` | 解析执行引擎并执行目标（Template 跳过） |
| `AssertionStep` | 运行断言规则 |
| `ExecutionEngineResolver` | 根据 Target.Type 选择 IExecutionEngine |
| `WorkflowJob` | Hangfire 作业：分布式锁 + 心跳 + Orchestrator |

### Execution — 执行引擎

| 引擎 | 技术 | 关键特性 |
|------|------|---------|
| `HttpExecutionEngine` | Flurl.Http | 认证(Bearer/Basic/ApiKey)、代理、SSL忽略、重试、限速 |
| `TcpExecutionEngine` | TcpClient + SslStream | TLS、消息收发、连接延迟 |
| `DbExecutionEngine` | ADO.NET | SqlServer/MySQL/PostgreSQL、Query/NonQuery/Scalar |
| `PythonExecutionEngine` | Process | 脚本/文件执行、超时终止、进程树清理 |

每种引擎同时实现 `IExecutionEngine`（给管线用）和 DSL 步骤执行器（`IStepExecutor`，给模板用）。

### Dsl — DSL 解析

| 类 | 职责 |
|----|------|
| `DslParser` | JSON → StepSequence，`{{var}}` 和 `{{var:default}}` 变量替换 |
| `DslSchemaValidator` | 校验 steps 数组存在且合法 |
| `TemplateResolutionStep` | 管线步骤：解析模板 DSL，创建 DslPipelineContext |
| `VariableResolver` | 运行时变量替换（第二步解析，与 DslParser 的正则一致） |

### Orchestration — DSL 运行时

| 类 | 职责 |
|----|------|
| `ExecutionEngine` | DAG 执行引擎：顺序步骤 + 并行组 + 重试退避 |
| `RuntimeOrchestrationStep` | 管线步骤：调用 DSL 引擎，汇总断言结果 |
| `CircuitBreaker` | 断路器：每目标键，5 次失败 / 60s 窗口 |
| `RedisProgressStore` | 进度持久化：Redis，6h TTL，支持崩溃恢复 |

### Infrastructure — 基础设施

| 组件 | 说明 |
|------|------|
| `MonitorRepository` | Dapper，TargetConfig JSON ↔ MonitorTarget 反序列化 |
| `ExecutionRecordRepository` | 执行记录 CRUD + 统计 + TopN 错误 |
| `DapperOutboxRepository` | Outbox 消息写入/锁定/标记 |
| `AnalysisRepository` | AI 分析结果持久化 |
| `AiTaskService` | AI 任务队列（乐观锁批处理） |
| `OutboxDispatcherHostedService` | 后台轮询 Outbox → MediatR 分发 |
| `AiWorker` | 后台轮询 AiTask → LLM 调用 → 存储 |
| `ExecutionWatchdogHostedService` | 检测僵死执行（心跳超时 2min / 总时长 15min） |
| `HangfireWorkflowScheduler` | Hangfire 调度封装 |
| `MonitorHub / LogHub` | SignalR 实时推送 |

### Auth — 认证授权

| 组件 | 说明 |
|------|------|
| `JwtTokenIssuer` | JWT 签发（sub/role/perm，8h 过期） |
| `DapperAuthService` | 登录/刷新/注销，PBKDF2 密码验证 |
| `Pbkdf2PasswordHasher` | PBKDF2-SHA256, 150k 迭代, 16B 盐 |
| `DapperPermissionStore` | RBAC 权限查询，MemoryCache 缓存 5min |
| `PermissionAuthorizationHandler` | ASP.NET Core 策略授权（`perm:xxx` 格式） |

## 公共组件

### CacheCommons
- `MemoryCacheService` — 防击穿（每 key 一个 SemaphoreSlim），防穿透（缓存空值），TTL 随机抖动

### LockCommons
- `RedisLock` — SET NX EX + 自动续期（TTL/2 间隔）+ Lua 安全释放
