# AutoTest — 项目概述

## 1.1 项目是什么

AutoTest 是一个**分布式系统监控与自动化回归测试平台**，面向微服务架构下的 API、TCP 端点、数据库查询和 Python 脚本执行场景。它解决了传统测试工具在分布式环境中无法有效应对的三大问题：

- **被动式告警太多** —— 传统监控只能告诉你"服务挂了"，不能告诉你"为什么挂"
- **排障链路割裂** —— 日志、指标、执行记录散落在不同系统，人工串连耗时巨大
- **回归覆盖难持续** —— 团队迭代快，手工构建的断言集很快就和实际接口对不上了

AutoTest 的核心思路是：**定时执行 → 自动断言 → 失败时 AI 分析根因 → 结构化输出修复建议**，形成一条从检测到分析的自动化闭环。

## 1.2 核心能力

| 能力 | 说明 | 优先级 |
|------|------|--------|
| 多协议监控 | HTTP / TCP / SQL / Python 脚本执行 | P0 |
| 灵活断言 | 状态码 / 响应体 / JSON Path / 自定义算子 | P0 |
| 定时调度 | 每日自动执行 + 执行次数限制 | P0 |
| 集中日志 | Elasticsearch 聚合 + SignalR 实时推送 | P0 |
| AI 根因分析 | LLM 驱动的错误分类 + 根因定位 + 修复建议 | P0 |
| 自动恢复 | 重试执行 / 配置修复建议 / 通知（Pipeline） | P1 |
| RBAC 权限 | 基于角色的访问控制（用户 / 角色 / 权限） | P1 |
| 实时看板 | 执行统计（成功率 / 趋势 / Top 错误） | P1 |

## 1.3 技术栈

| 层面 | 技术选型 |
|------|----------|
| 运行时 | .NET 8+, ASP.NET Core |
| 数据库 | SQL Server / SQLite（双 Provider） |
| 日志 | Serilog + Elasticsearch |
| AI 推理 | Semantic Kernel + 豆包 / OpenAI 兼容模型 |
| 任务调度 | Hangfire |
| 事件总线 | MediatR + Outbox Pattern |
| 缓存 | MemoryCache + Redis（可选） |
| 实时推送 | SignalR |
| ORM | Dapper |
| 迁移 | FluentMigrator |
| 认证 | JWT + RBAC（自定义 Dapper 实现） |

## 1.4 适用场景

- **API 回归测试**：对关键接口定时执行 HTTP 健康检查，自动验证响应结构
- **TCP 服务监控**：检测消息队列 / 中间件端口的连通性与响应
- **数据库巡检**：定时执行 SQL 查询并断言结果范围
- **自定义脚本检测**：通过 Python 脚本实现灵活的业务级监控
- **故障根因分析**：执行失败后自动聚合日志，由 LLM 定位根因
- **值班告警**：分析结果通过结构化 JSON 输出，可对接钉钉/飞书等通知渠道
